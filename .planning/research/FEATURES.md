# Features Research — What Makes OSS Developer Tools Go Viral

## Table Stakes (must have or developers skip)

| Feature | Why | Evidence |
|---------|-----|----------|
| **One-command install** | Friction kills adoption. `npx` or `claude plugin add` must just work | sniff gets installs from `npx sniff-qa` simplicity |
| **Compelling README** | 88% of users form opinion in seconds. README IS the product page | GitHub data: repos with good READMEs get 3-5x more stars |
| **Real output example** | Developers need to see what they get BEFORE installing | Top OSS tools show terminal output or screenshots |
| **MIT License** | Maximum adoption. No corporate legal friction | Most starred JS tools are MIT |
| **Zero config** | Works out of the box on any project. No setup required | sniff pattern: "That's the whole tutorial" |
| **Clear badges** | npm, CI, license, node version. Signals maintained, professional | Standard across all credible OSS |
| **CONTRIBUTING.md** | Shows the project welcomes contributions. Community signal | Required for GitHub "Community Profile" score |

## Differentiators (competitive advantage for stars/adoption)

| Feature | Why | Evidence |
|---------|-----|----------|
| **Quantified scores (1-10)** | Gamification. Developers love metrics. "My project scores 7.2/10" | Lighthouse scores went viral because of this pattern |
| **Evidence-based findings** | Not opinions — research citations with confidence levels | 30+ quantified findings in evidence-base.md |
| **12-dimension analysis** | Comprehensive. No other tool audits across this many axes | Unique in the Claude Code plugin ecosystem |
| **2026-aware scoring** | Detects modern CSS (View Transitions, scroll-driven, container queries, OKLCH) | Cutting-edge signals that competing tools miss |
| **19 knowledge documents** | Built-in design expertise. Not just rules — principles, psychology, trends | 3,081 lines of curated knowledge |
| **ASCII art branding** | Memorable. sniff's cat got attention. Need equivalent visual identity | sniff pattern proven effective |
| **Before/after code** | Actionable. Shows exactly what to change | Top finding format in the scoring engine |
| **Framework detection** | Auto-detects React, Vue, Svelte, Next, Tailwind, shadcn. No config needed | Frictionless experience |

## Anti-Features (deliberately NOT doing)

| Anti-Feature | Why NOT |
|--------------|---------|
| Web dashboard | CLI-first philosophy. Complexity kills adoption |
| Paid tier | Pure OSS builds trust and adoption faster |
| Auto-fix | Read-only analysis is safer. No risk of breaking user code |
| Config files | Zero config is the feature. Don't make users configure |
| Figma integration | Scope creep. Claude Code plugin first |
